# PWA (Progressive Web App) Features

Про Запас is now a fully functional Progressive Web App (PWA) that can be installed on your device and used offline!

## Features

### 📱 Installable
- Add the app to your home screen on mobile devices
- Pin to taskbar on desktop browsers
- Full-screen experience without browser chrome

### 🔄 Offline Support
- Service Worker caches essential assets
- Offline fallback page when connection is lost
- Cached API responses for faster loading

### ⚡ Auto-Updates
- Automatic service worker updates when new versions are available
- User prompt to reload when updates are ready
- Seamless background updates

### 🎨 Native-like Experience
- Custom app icons (192x192 and 512x512)
- Splash screen with app branding
- Theme color matching app design

## Installation

### iOS (Safari)
1. Open the app in Safari
2. Tap the Share button
3. Select "Add to Home Screen"
4. Tap "Add"

### Android (Chrome)
1. Open the app in Chrome
2. Tap the "Add to Home Screen" banner when it appears
3. Or tap the menu (⋮) > "Install app"

### Desktop (Chrome/Edge)
1. Click the install icon in the address bar
2. Click "Install"
3. The app will open in a standalone window

## Development

### Generate App Icons
```bash
npm run generate-icons
```

This generates PNG icons from the SVG favicon in the correct sizes.

### Build for Production
```bash
npm run build
```

The build process automatically:
- Generates the service worker
- Precaches all static assets
- Creates the web manifest
- Optimizes for production

### Test PWA Locally
```bash
npm run build
npm run preview
```

> **Note:** Service workers only work in production builds and over HTTPS (or localhost).

## Service Worker Strategy

The app uses **injectManifest** strategy with custom service worker (`src/sw.ts`):

- **Precaching**: All static assets (JS, CSS, HTML, icons)
- **Runtime Caching**:
  - Google Fonts: CacheFirst (1 year)
  - API endpoints: NetworkFirst (1 hour)
  - Navigation: NetworkFirst with offline fallback

## Configuration

The PWA is configured in `vite.config.ts` using `vite-plugin-pwa`:

- **strategies**: `injectManifest` for custom service worker
- **registerType**: `prompt` to ask users before updating
- **manifest**: App name, colors, icons, and metadata
- **workbox**: Caching strategies and expiration policies

## Testing PWA Features

### Chrome DevTools
1. Open DevTools > Application tab
2. Check "Service Workers" for registration status
3. Check "Cache Storage" for cached assets
4. Use "Offline" checkbox to test offline mode

### Lighthouse
Run a Lighthouse audit to verify PWA compliance:
- ✅ Uses HTTPS
- ✅ Registers a service worker
- ✅ Responds with 200 when offline

- ✅ Manifest includes name, icons, start_url, display

## Troubleshooting

### Service Worker Not Registering
- Make sure you're using HTTPS or localhost
- Clear browser cache and reload
- Check browser console for errors

### App Not Installable
- Verify manifest is loading correctly
- Check that icons are accessible
- Ensure HTTPS is enabled (required for installation)

### Offline Not Working
- Service worker may not be installed yet - reload the page
- Check DevTools > Application > Service Workers
- Try unregistering and reloading
